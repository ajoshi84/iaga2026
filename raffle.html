GitHub Connection

The app saves raffle-history.json to the root of your GitHub repo on every save. Data is loaded from GitHub on startup and always wins over local cache.

Generate at GitHub → Settings → Developer settings → Personal access tokens → Fine-grained. Needs Contents: Read & Write on your repo. Token is stored only in your browser's localStorage.

How it works

On load — fetches raffle-history.json from GitHub. If found, it replaces any local data.

On save — every time a winner is confirmed or a manual entry is added, the file is written back to GitHub automatically.

Offline fallback — if GitHub is unreachable, the app falls back to localStorage so you never lose data mid-raffle.

Multi-device — anyone with the app open (e.g. on a laptop at the course) will load the same history since it reads from GitHub each time.
